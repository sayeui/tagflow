### 项目结构与功能概述

根据提供的代码文件，以下是项目的结构和主要功能的概述：

#### 1. 配置文件 
- **`config.yml`**:
  - 提供更详细的配置选项，包括是否使用MinIO、S3存储桶名称、区域名称以及MinIO的端点URL。

#### 2. 文件管理器 (`file_manager.py`)
- **类 `FileManager`**:
  - 负责与对象存储（如AWS S3或MinIO）进行交互。
  - 主要功能包括上传文件、删除文件、计算文件哈希值以及检查文件是否存在。
  
#### 3. 文件夹导入器 (`folder_importer.py`)
- **类 `FolderImporter`**:
  - 用于将本地文件夹中的所有文件递归地上传到对象存储，并为每个文件自动添加标签。
  - 支持删除文件夹中的所有文件及其子目录。

#### 4. 标签管理器 (`tag_manager.py`)
- **类 `TagManager`**:
  - 管理文件的标签，支持自动标记、添加标签、移除标签和获取文件标签。
  - 使用Pandas DataFrame来存储文件路径和对应的标签列表。

#### 5. 应用程序入口 (`app.py`)
- **Flask应用**:
  - 提供多个API端点，用于文件上传、标签管理和文件夹导入等功能。
  - 使用Flask-APISpec生成API文档，并提供Swagger UI接口。
  - 支持通过命令行参数指定配置文件路径，默认为`config.yml`。

#### 6. API文档 (`api_documentation_zh.md` 和 `api_documentation_en.md`)
- **中文和英文版本**:
  - 描述了API的主要端点及其请求和响应格式。
  - 包括上传文件、获取文件标签、从对象存储获取文件、导入文件夹到对象存储、为文件添加标签和从文件移除标签等操作。

### 总结
该项目实现了一个基于Flask的文件管理系统，提供了文件上传、标签管理和文件夹导入等功能。通过配置文件可以灵活选择使用AWS S3或MinIO作为对象存储服务。API文档详细描述了各个端点的功能和使用方法，支持多语言（中文和英文）。
